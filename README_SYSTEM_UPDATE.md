# 小车自动寻球系统改进说明

## 📋 改进总结

### 原系统
- ✓ 场上 1 个小球，位置随机生成
- ✓ 小车初始位置随机生成  
- ✓ 小车找到球后直接移动到三角形区域

### 新系统
- ✓ 场上 **9 个小球**，位置**固定**
- ✓ 小车初始位置**固定**
- ✓ 小车需找到 **3 个球**后才能移动到三角形区域
- ✓ 每个球只能收集一次

---

## 🔧 实现细节

### 代码修改

**CarAgent.cs 主要改变**：

1. **字段改变**
   ```csharp
   // 旧：单个球
   public Transform ball;
   
   // 新：9个球的数组 + 固定位置
   public Transform[] balls;
   public Vector3 carFixedSpawnPosition;
   public Vector3[] ballFixedSpawnPositions;
   ```

2. **新增追踪机制**
   ```csharp
   private int ballsCollected = 0;      // 已收集球数
   private bool[] ballCollected;        // 每个球的收集状态
   private const int BALLS_TARGET = 3;  // 目标：3个球
   ```

3. **核心方法改变**
   - `BallFound()` → `CollectBall(int ballIndex)`: 收集指定球
   - `StartMoveToTriangle()`: 当收集满 3 个球时调用
   - `OnEpisodeBegin()`: 改为使用固定位置而非随机生成
   - `OnActionReceived_FindBall()`: 改为找离车最近的未收集球

4. **碰撞检测**
   - 碰撞时会查找碰撞的是哪个球，调用 `CollectBall(i)`

## 📝 配置步骤

### 最简单的配置方式：直接在编辑器拖拽

1. **在场景中放置9个小球到你想要的位置**
   - 在 Hierarchy 中复制小球8次，共9个
   - 拖动它们在 Scene 视图中放到相应位置
   - 为每个小球添加 "Ball" 标签

2. **设置小车初始位置**
   - 在 Scene 视图中拖动小车到起始位置
   - 或在 Inspector 中设置 Transform > Position

3. **配置 CarAgent 脚本**
   - 选择小车对象，找到 CarAgent 脚本
   - Balls 数组大小改为 9
   - 将9个小球从 Hierarchy 中拖入到 Ball[0] ~ Ball[8]

就这样！系统会自动记录初始位置，每局重新恢复。

## 🎮 核心流程

```
游戏开始
  ↓
小车出现在固定位置
9个球出现在固定位置
  ↓
小车开始寻找球（自动导航或 RL 控制）
  ↓
收集到球 1 → 隐藏，计数器: 1/3
  ↓
收集到球 2 → 隐藏，计数器: 2/3
  ↓
收集到球 3 → 隐藏，计数器: 3/3
  ↓
状态转换：FindBall → MoveToTriangle
小车自动导航到三角形中心
  ↓
小车进入三角形区域
等待 3 秒
  ↓
成功！回合结束，重新开始
```

## 🔍 调试技巧

### Console 日志输出

运行游戏时会显示：
- `小车初始位置已设置：...` - 小车位置确认
- `球i位置已设置：...` - 每个球的位置（共9条）
- `收集到第 X 个球` - 收集进度
- `成功收集 3 个球，开始导航...` - 转换到三角形阶段

### 常见问题

**Q: 小车看不到球**
- A: 检查摄像头的视角范围；增加球与小车的距离；调整 RayPerception 传感器参数

**Q: 球没有被收集**
- A: 确认球有 "Ball" 标签；检查碰撞体设置；确保球在数组中

**Q: 小车找到球后不移动到三角形**
- A: 确保 ballsCollected 计数器达到 3；检查状态转换逻辑；查看控制台日志

**Q: 位置不在场景内**
- A: 确保 Y 坐标大于地面；X、Z 坐标在地板范围内；调整 spacing 参数
